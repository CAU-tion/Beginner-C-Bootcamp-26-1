## Challenge1. Format String Bug 실습

### 실습 내용
취약한 코드 `printf(argv[1]);`를 작성하고 컴파일한 뒤,  
포맷 문자열(`%x`, `%p`, `%d`, `%s`)을 입력하여 출력 결과를 확인하였다.  
이후 안전한 코드 `printf("%s", argv[1]);`로 수정하여 결과를 비교하였다.

### 핵심 개념
`printf`는 포맷 지정자에 따라 **메모리에 저장된 값을 해석하는 방식이 달라지는 함수**이다.

- `%x` : 정수를 16진수로 출력  
- `%p` : 메모리 주소 출력  
- `%d` : 정수 출력  
- `%s` : 문자열 출력  

사용자 입력을 그대로 `printf(argv[1]);`에 전달하면 입력값이 문자열이 아닌 **포맷 문자열로 해석된다.**  
이 경우 존재하지 않는 인자를 스택 메모리에서 읽어 출력하게 되어 취약점이 발생한다.

### 실습 결과
- `%x`, `%p`, `%d` 입력 시 스택의 임의 값이 출력되었다.  
- `%s` 입력 시 메모리를 문자열로 해석하여 깨진 문자열이 출력되었다.  
- 안전한 코드에서는 입력 문자열이 그대로 출력되며 포맷 지정자가 해석되지 않았다.  

### 원인
`printf(argv[1]);`는 사용자 입력을 데이터가 아닌 포맷 문자열로 해석한다.  
따라서 `%x`, `%p` 등의 지정자를 만나면 추가 인자가 있다고 가정하고 스택 메모리에서 값을 읽는다.  
하지만 실제로는 인자가 전달되지 않았기 때문에 임의의 메모리 값이 출력된다.

## Challenge2. 반복문 3종 비교

### 실습 내용
1부터 10까지의 합을 구하는 프로그램을  
`for`, `while`, `do-while` 반복문으로 각각 구현하고 실행 결과를 비교하였다.

### 핵심 개념
반복문은 같은 작업을 여러 번 수행할 때 사용하는 문법이다.

- `for` : 초기값, 조건식, 증감식을 한 줄에 작성  
- `while` : 조건식 중심으로 반복  
- `do-while` : 실행 후 조건을 검사하는 구조  

### 실습 결과
세 반복문 모두 `sum = 55`로 동일한 결과가 출력되었다.

### 코드 차이점 및 사용 상황

- **for**
  → 초기화, 조건, 증가가 한 줄에 있어 구조가 명확  
  → 반복 횟수가 정해져 있을 때 적절  

- **while**
  → 조건만으로 반복을 제어  
  → 반복 횟수보다 “조건 유지 여부”가 중요할 때 적절  

- **do-while**
  → 최소 1번 실행 보장  
  → 반드시 한 번은 실행해야 하는 경우 적절

## Challenge3. 조건문으로 분기 구현

### 실습 내용
사용자로부터 정수를 입력받아 해당 값이 양수, 음수, 0인지 판별하는 프로그램을  
`if`, 삼항 연산자, `switch` 방식으로 각각 구현하였다.

### 핵심 개념
조건문은 조건에 따라 서로 다른 코드를 실행하기 위한 문법이다.

- `if` : 범위 조건 처리 가능  
- `switch` : 특정 값 비교에 적합  
- 삼항 연산자(`?:`) : 간단한 조건을 한 줄로 표현  

### 실습 결과 및 정리
입력값에 따라 양수, 음수, 0이 정상적으로 출력되었다.

- `if` → 범위 조건 처리에 가장 적합  
- 삼항 연산자 → 간단한 조건을 간결하게 표현 가능하지만 가독성이 떨어질 수 있음  
- `switch` → 범위 비교가 어려워 상태값으로 변환하여 사용

## Challenge4. gdb로 반복문 뜯어보기

### 실습 내용
`for_sum.c`를 `gcc -g -O0` 옵션으로 컴파일한 뒤 gdb로 실행하여  
프로그램의 실행 흐름과 내부 상태를 확인하였다.

### 핵심 개념
gdb는 프로그램을 한 줄씩 실행하며 내부 동작을 분석할 수 있는 디버깅 도구이다.  
변수 값, 레지스터, 메모리 상태를 직접 확인할 수 있다.

- `break` : 중단점 설정  
- `run` : 프로그램 실행  
- `list` : 현재 코드 위치 확인  
- `next` : 다음 줄 실행  
- `step` : 함수 내부 진입  
- `print` : 변수 값 확인  
- `info registers` : 레지스터 확인  
- `x/` : 메모리 내용 확인  

### 실습 결과
- `next` 실행 시 반복문이 진행되면서 같은 줄이 반복 실행됨을 확인하였다.  
- `print i`, `print sum`을 통해 변수 값이 증가하고 누적되는 과정을 확인하였다.  
- `info registers`를 통해 CPU 레지스터 값을 확인하였다.  
- `x/` 명령어를 통해 스택 메모리 값을 확인하였다.  
- `step` 실행 시 `printf` 내부 진입을 시도하였으며, 라이브러리 소스가 없어 경고가 발생하였다.
