Challenge-1

취약한 코드를 컴파일 한 후 ./vuln "%x %x %x %x"를 명령어로 입력할 시
argv[0] = "./vuln"
argv[1] = "%x %x %x %x"
argv에 위와 같은 값이 담기게 된다.

여기서 printf(argv[1])이 실행되는데 이 때, printf("%x %x %x %x")로 함수가 실행되며,
입력값이었던 "%x %x %x %x"를 형식지정자로 인식하게 된다. 따라서 printf 함수가 실행되는 시점의 스택 프레임 바로 아래에 위치한 스택 내부의 의도치 않았던 값이 출력되고 이로 인해 메모리 주소가 유출 될 수 있으며 프로그램 조작 가능성 또한 생기게 된다.

만약 안전한 코드를 컴파일 한 후 위와 같은 명령어를 입력한다면, argv 배열에는 위와 똑같이 값이 담기게 되지만
형식지정자가 명시되어 있기에, 입력값인 "%x %x %x %x"를 단순 문자열로 인식하여, "%x %x %x %x"를 동일하게 출력하게 된다.

Challenge-2

for: 가독성이 가장 좋고, 반복 횟수가 명확할 때 사용 유리

while: 조건식만 체크하며, 조건이 거짓이면 한 번도 실행 안 됨. 반복 횟수가 불확실하고, 특정 조건이 만족될 때까지 반복문이 돌아가야할 때 사용 유리

do-while: 조건을 마지막에 체크, 무조건 최소 1회는 실행되며 입력을 먼저 받고 그 값에 따라 반복 여부를 결정할 때 사용 유리
